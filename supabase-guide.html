Your SQL Guide

Everything you need to manage your posts directly in Supabase. No code. Just open, type, run.

Step 1 — Open Supabase SQL Editor

Navigate

How to reach the SQL editor

1. Go to supabase.com → sign in → click your project pwafyokcjjqejgsghvmf

2. In the left sidebar → click "SQL Editor" (looks like </>)

3. Click "New Query" in the top left

4. Type your SQL → hit Run (or Ctrl+Enter)

Your table is called: posts — everything lives here.

Your Table Structure

ColumnTypeWhat it is
iduuidAuto-generated unique ID — never touch this
titletextThe post title
slugtextURL-friendly version of title (lowercase, dashes)
languagetext'English' or 'বাংলা'
categorytextEssay, Book Review, Commentary, Personal, প্রবন্ধ
datedateFormat: YYYY-MM-DD
excerpttext2–3 line preview shown on cards
contenttextFull post body. Use blank lines for paragraphs, > for quotes
image_urltextURL to cover image (from Supabase storage or anywhere)
featuredbooleantrue = shows as big hero card. Only one should be true.
created_attimestampAuto-set when created

See All Your Posts

SELECT id, title, language, category, date, featured
FROM posts
ORDER BY date DESC;
→ Shows a table of all posts sorted newest first
-- See everything including full content
SELECT *
FROM posts
WHERE title LIKE '%Partition%';
→ Finds any post with "Partition" in the title

Add a New Post

Copy this, fill in your content, run it:

INSERT INTO posts (title, slug, language, category, date, excerpt, content, featured)
VALUES (
  'My New Essay Title',
  'my-new-essay-title',       -- same as title but lowercase + dashes
  'English',                    -- or 'বাংলা'
  'Essay',                      -- Essay, Book Review, Commentary, Personal, প্রবন্ধ
  '2026-03-12',                 -- today's date in YYYY-MM-DD
  'Short 2-line preview that shows on the card.',
  'First paragraph of your essay.

Second paragraph here. Use blank lines between paragraphs.

> This is a pull quote — it will be styled in red.

Continue writing here.',
  false                         -- set true if you want this as the featured post
);
For বাংলা posts: set language = 'বাংলা' and category = 'প্রবন্ধ'

Edit an Existing Post

First find the post ID:

SELECT id, title FROM posts WHERE title LIKE '%Farewell%';

Then update using that ID:

UPDATE posts
SET
  title = 'A Farewell to Arms — Updated',
  excerpt = 'New excerpt here.',
  content = 'New full content here.'
WHERE id = 'paste-the-uuid-here';
Tip: You can update just one field — just list the ones you want to change after SET.

Change the Featured Post

-- Step 1: un-feature everything
UPDATE posts SET featured = false;

-- Step 2: feature the one you want
UPDATE posts
SET featured = true
WHERE title = 'The True Cost of Partition';
→ This post now shows as the big hero card on your homepage

Delete a Post

-- First check you have the right one:
SELECT id, title FROM posts WHERE title = 'Post Title Here';

-- Then delete it:
DELETE FROM posts WHERE id = 'paste-uuid-here';
Always SELECT first, then DELETE. There's no undo in SQL.

Seed All 12 Writings at Once

If your site shows posts from the local fallback but not from Supabase — just go to admin.html → click "↓ Seed All Writings" in the left nav. It inserts all 12 posts automatically, skipping any that already exist.

Or run this to check how many posts are in Supabase right now:

SELECT COUNT(*) FROM posts;
→ If this returns 0, use the Seed button in admin.html

Useful Queries to Keep

All English posts

SELECT title, category, date FROM posts
WHERE language = 'English'
ORDER BY date DESC;

All Bengali posts

SELECT title, category, date FROM posts
WHERE language = 'বাংলা'
ORDER BY date DESC;

Posts without images

SELECT title FROM posts
WHERE image_url IS NULL;

Add an image URL to a post

UPDATE posts
SET image_url = 'https://your-image-url.com/photo.jpg'
WHERE title = 'Norwegian Wood';

Change a category

UPDATE posts
SET category = 'Personal Essay'
WHERE id = 'uuid-here';

Count by category

SELECT category, COUNT(*) as total
FROM posts
GROUP BY category
ORDER BY total DESC;
Remember: Changes in Supabase SQL Editor affect your live site immediately. The admin panel at admin.html also writes directly to Supabase — use whichever feels easier for the task.
